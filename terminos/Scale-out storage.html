Scale-out storage (Almacenamiento escalable)
La escalabilidad es un aspecto que tiene que ver con el largo plazo. Es decir, cuánta capacidad tiene la herramienta SaaS que utilizas para poder satisfacer tus necesidades, o si por el contrario, te obligaría a cambiar de herramienta con el tiempo porque no dispone de recursos suficientes en espacio de almacenamiento para abastecerte. El almacenamiento escalable es un enfoque de almacenamiento en el que varias máquinas más pequeñas, o nodos, se conectan y configuran para actuar como una unidad lógica. Cuando un sistema de escalamiento horizontal se acerca a su límite de almacenamiento, se puede agregar fácilmente otro nodo para aumentar la capacidad de almacenamiento. Como el cliente del sistema de almacenamiento de objetos se dirige a una dirección FQDN virtual, es sencillo agregar más nodos al clúster, el software de almacenamiento de objetos incluye los nuevos nodos en el clúster y redistribuye los datos entre estos nodos. Por cada nuevo nodo agregado, agrega no solo capacidad de almacenamiento de disco adicional (y rendimiento), sino también RAM, CPU y recursos de red adicionales para mejorar el rendimiento general del clúster. Este método de escalado evita las limitaciones de la arquitectura de almacenamiento de escalamiento vertical tradicional en la que cada IO tiene que ser procesado por uno de solo dos controladores de almacenamiento.
